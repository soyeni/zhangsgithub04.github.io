You can also run the repository locally on your own machine, which requires the skill to be able to setup the environement (see below)
Install Anaconda
git clone the repository to your local computer
create a virtual environment for your project using conda
pip install -r requirement.txt
possibly fix the requirement.txt file for those packages that conflict or are outdated, conservatively, increamentally.
This process might not be as pleasant as running in colab.
If there are jupyter notebook version, then they should be able to make running on colab as well
To clone the repository directly to your colab, you can run git in a code cell. Pay attention to ! operator. !git clone https://github.com/yourusername/yourpublicrepo.git This will create a folder called yourpublicrepo.
