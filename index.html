You can also run the repository locally on your own machine, which requires the skill and effort to setup the environement (see below)
  1. Install Anaconda
  2. git clone the repository to your local computer
  3. Enter into the directory you just cloned
  4. look for environment.yml file under the directory. read the content inside. This file specify the packages with compatible versions needed for experimenting. Will be used in the next step!!!
  5. create a virtual environment for your project using conda from the environment.yml file, where the first line sets the new environment's name.
  6. conda env create -f environment.yml
  7. conda env list
  8. Verify that the new environment was installed correctly. The above should show you a list of virtual environment created so far including the one just created. What is the name of the new environment just created? read again the environment.yml file, where the first line sets the new environment's name. Make sure the environment name is replaced with the correct one in the next command for different environment!!!
  9. Activate the new environment: conda activate binder_env
  10. Now you can run Python training.py ... whatever .py file could be there that is supposed to be used. Have fun!
  11. after you have used the virual environment for exploring your project, exit the environment. conda deactivate
  12. For details see Creating an environment.
No need to try the following, assuming you followed the above procedure (which is better than the requirements.txt approach!!!
pip install -r requirements.txt
possibly fix the requirements.txt file for those packages that conflict or are outdated, conservatively, increamentally.
